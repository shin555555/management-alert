## フェーズ6: 既存在籍者の一括登録機能 (Phase 6)

> **背景**: 現行の新規登録フローは「これから利用を開始する人」向けに設計されており、開始月を起点にタスク日程を自動生成する。しかし、システム導入時点で既に事業所に在籍している利用者（以下「既存在籍者」）は、各タスクが既にサイクルの途中であるため、自動生成された日程とは実態が乖離する。本フェーズでは、既存在籍者を正確にシステムへ取り込むための「途中登録モード」を実装する。

### 課題の整理
- 通常の新規登録：「利用開始月」→ 全タスクの開始日・終了日を自動計算（サイクル先頭から）
- **既存在籍者の登録**：各タスクは既にサイクルの途中 → 現在の期の実際の開始日・終了日を手入力する必要がある

### 実装タスク
- [x] **途中登録ダイアログの追加**（`src/app/clients/existing-client-dialog.tsx`）
  - 利用者の基本情報（氏名・利用開始日・メモ）を入力するステップ1
  - テンプレート一覧を表示し、各項目の「現在の期の開始日・終了日・現在ステータス」を個別入力するステップ2（DateInput コンポーネント流用）
  - 自動計算は行わず、入力値をそのままタスクとして登録する
- [x] **利用者一覧ページへの導線追加**（`src/app/clients/page.tsx`）
  - 既存の「新規登録」ボタンに加え「既存在籍者を追加」ボタンを設置
- [x] **Server Action の追加**（`src/app/clients/actions.ts`）
  - 途中登録専用の `registerExistingClient` アクション：自動日程計算をスキップし、入力値でタスクを直接生成
- [x] **スキップ可能なテンプレート対応**
  - 既存在籍者の場合、一部のタスクが「現在の期は不要（次回更新から管理開始）」となるケースがある
  - 各テンプレートに「この期はスキップ（次回サイクルから開始）」チェックボックスを設ける

## フェーズ7: バグ修正・UX改善 (Phase 7)

> **背景**: 実際にシステムを運用してみて発見された不具合・使いにくさを修正する。ユーザーフィードバックに基づく改善フェーズ。

### 7-1. ダッシュボード表示バグ修正

- [x] **ダッシュボードにステータス進行中のタスクが表示されない問題の修正**
  - **現象**: 利用者詳細画面では「利用者負担の適用期間」が「未対応」でステータス進行ボタンが表示されているが、ダッシュボードの「対応待ちタスク」に何も表示されない
  - **原因**: `getDashboardData()` のフィルタリングで、ステータスが「未対応」（最初のステータス）かつアラートレベルなし（期限が十分先）の場合は除外される。しかしユーザー視点では「未対応のまま放置されているタスク」もダッシュボードで把握したい
  - **対応方針**: ダッシュボードの表示条件を見直し、「全未完了タスク」または「未対応タスクも含む」オプションを検討する

### 7-2. 利用者タスク編集機能の追加

- [x] **登録済みタスクの後から編集できる機能を追加**
  - **現象**: 利用者登録時に一部の情報が不明で飛ばした場合、後から設定する手段がない
  - **要件**:
    - 利用者詳細画面から各タスクの開始日・終了日・ステータスを編集できるUIを追加
    - 未入力（スキップ）だったタスクを後から有効化できる機能
    - 編集時は TaskHistory に変更履歴を記録する

### 7-3. 日付入力の形式拡張

- [x] **「令和5年2月1日」形式（日本語表記）での入力に対応する**
  - **現象**: 受給者証に記載されている日付は「令和5年2月1日」形式。現状はこれを「R50201」や「20230201」に手動で変換する必要がありストレスが大きい
  - **対応する入力形式**（既存 + 新規追加）:
    - 既存: `R90228`, `090228`, `20270228`, `2027-02-28`
    - **新規**: `令和5年2月1日`, `令和5年2月01日`, `R5年2月1日` など日本語表記
  - **修正対象**: `src/lib/wareki.ts` の `parseWarekiInput()` 関数

### 7-4. 和暦パースのバリデーションバグ修正

- [x] **`R50201` 入力で「桁数が正しくありません」エラーが出るバグの修正**
  - **現象**: 「R50201」（令和5年2月1日）と入力すると、パースエラーが発生する
  - **原因**: `parseWarekiInput()` で "R" を除去した後の数字 "50201" が5桁。`parseReiwaDigits()` は6桁（年2桁+月日4桁）か7桁（年3桁+月日4桁）しか受け付けないため、年1桁+月日4桁の5桁パターンがエラーになる
  - **修正**: `parseReiwaDigits()` で5桁（年1桁+月日4桁）パターンを追加する

## フェーズ8: ダッシュボードUX改善 (Phase 8)

> **背景**: タスク数が増えるとフラットなリスト表示が煩雑になるため、期限月ごとにグループ化し折りたたみ可能なUIに改修する。

### 8-1. 期限月グループ化表示

- [x] **タスクを期限月ごとにグループ化して表示**
  - タスクを `endDate` の年月でグループ化し、和暦（令和○年○月）のセクションヘッダーで区切る
  - 各セクションは折りたたみ可能（直近2ヶ月はデフォルト展開、それ以降は折りたたみ）
  - 折りたたみ状態でもグループヘッダーにサマリーバッジ（緊急/警告/準備/未対応/進行中の件数）を表示
  - グループヘッダーの左ボーダー色は最も緊急度の高いアラートレベルで決定
  - 「すべて展開」「すべて折りたたみ」ボタンを追加
