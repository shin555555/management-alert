# 開発マイルストーン・ロードマップ

開発を効率的に進めるためのステップを定義する。
AIエージェントは、このロードマップに沿って段階的に実装を行う。

## フェーズ1: 基盤構築とスキーマ定義 (Phase 1)
- [ ] プロジェクトの初期化（Next.js等のセットアップ）
- [ ] TailwindCSS, UIライブラリ(Shadcn UI等)の導入
- [ ] データベースの設定とPrismaスキーマの構築 (`docs/02_database_schema.md`に基づく)
- [ ] 基本的なルーティングとレイアウト（サイドバーナビゲーション等）の作成

## フェーズ2: マスタ管理機能と日付ロジック (Phase 2)
- [ ] 共通日付計算ユーティリティの実装（加算、固定、反復、月末丸め処理）
- [ ] 爆速入力UIのためのカスタムInputコンポーネント（和暦パース、全選択、上下キー操作）の実装
- [ ] 「A型事業所基本セット」のシードデータ(Seed)作成
- [ ] 事業所管理者がルール（TaskTemplates）を編集できる設定画面の実装

## フェーズ3: 利用者管理とタスク生成 (Phase 3)
- [ ] 利用者一覧と新規登録画面の実装
- [ ] 新規登録時の「スマート予測入力」の実装（開始月からの自動展開）
- [ ] 利用者詳細画面における、個別タスクの状態表示と手動修正UI

## フェーズ4: ダッシュボードとステータス管理 (Phase 4)
- [ ] ダッシュボード画面の構築（アラート対象、進行中タスクの抽出）
- [ ] 各種アラートレベル（黄・橙・赤）に応じたUIの切り替え（色の変化）
- [ ] ステータスのワンクリック進行機能（進捗フローに沿った更新）
- [ ] 完了時の「次回更新（ルーティン生成）」ロジックの実装

## フェーズ5: セキュリティ・権限・アーカイブ (Phase 5)
- [ ] 認証機能の導入（管理者/一般スタッフの権限分離）
- [ ] 利用者の退所処理（アーカイブ）機能と履歴閲覧ビューの実装
- [ ] UI/UXの最終調整（スマートフォン対応、レスポンシブ崩れの修正）
