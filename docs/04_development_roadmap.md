## フェーズ6: 既存在籍者の一括登録機能 (Phase 6)

> **背景**: 現行の新規登録フローは「これから利用を開始する人」向けに設計されており、開始月を起点にタスク日程を自動生成する。しかし、システム導入時点で既に事業所に在籍している利用者（以下「既存在籍者」）は、各タスクが既にサイクルの途中であるため、自動生成された日程とは実態が乖離する。本フェーズでは、既存在籍者を正確にシステムへ取り込むための「途中登録モード」を実装する。

### 課題の整理
- 通常の新規登録：「利用開始月」→ 全タスクの開始日・終了日を自動計算（サイクル先頭から）
- **既存在籍者の登録**：各タスクは既にサイクルの途中 → 現在の期の実際の開始日・終了日を手入力する必要がある

### 実装タスク
- [x] **途中登録ダイアログの追加**（`src/app/clients/existing-client-dialog.tsx`）
  - 利用者の基本情報（氏名・利用開始日・メモ）を入力するステップ1
  - テンプレート一覧を表示し、各項目の「現在の期の開始日・終了日・現在ステータス」を個別入力するステップ2（DateInput コンポーネント流用）
  - 自動計算は行わず、入力値をそのままタスクとして登録する
- [x] **利用者一覧ページへの導線追加**（`src/app/clients/page.tsx`）
  - 既存の「新規登録」ボタンに加え「既存在籍者を追加」ボタンを設置
- [x] **Server Action の追加**（`src/app/clients/actions.ts`）
  - 途中登録専用の `registerExistingClient` アクション：自動日程計算をスキップし、入力値でタスクを直接生成
- [x] **スキップ可能なテンプレート対応**
  - 既存在籍者の場合、一部のタスクが「現在の期は不要（次回更新から管理開始）」となるケースがある
  - 各テンプレートに「この期はスキップ（次回サイクルから開始）」チェックボックスを設ける
