# UI/UX・機能要件詳細定義

## 1. 概要
本ドキュメントは、要件定義書で指定された「爆速UI」「現場特化UI」を実現するための、具体的なフロントエンドの挙動と実装方針をまとめる。

## 2. 爆速入力UIの実現

### 2.1 スマート予測入力（新規登録時）
* **挙動**: 利用者の新規登録画面で「利用開始月」を選択すると、バックエンド経由またはフロントエンドのロジックで`TaskTemplates`（基本セット）のルールが評価され、全項目の開始日・終了日の初期値が一括生成される。
* **UI**: 生成された値は入力フォームにセットされた状態で表示され、ユーザーはスクロールしながら目視確認する。

### 2.2 イレギュラー期間の手動上書きUI（必須要件）
* **和暦（令和）の直接入力**:
  * 日付入力フィールド（Input）に対し、カスタムのパース処理を実装する。
  * ユーザーが `R90228` や `090228` と入力し、フォーカスが外れた（onBlur）瞬間に、内部で `2027-02-28` に変換し、画面表示は「令和9年2月28日」または「2027/02/28」にフォーマットする。
* **ワンクリック全選択**:
  * Inputコンポーネントの `onFocus` イベントで `e.target.select()` を発火させ、入力済みのテキストを即座にハイライトさせる。
* **キーボード操作による増減**:
  * `onKeyDown` イベントを捕捉し、「↑」キーで+1日、「↓」キーで-1日の日付操作を行う処理を実装する。

## 3. ダッシュボード（タスク管理）とアラート

* **表示ロジック**:
  * 期限が近づいているタスク（アラート判定）、または「進行中（未対応以外、完了未満）」のタスクのみをリスト表示。
  * 各タスク行には「利用者名」「項目名」「期限日」「現在ステータス変更セレクト/ボタン」を配置。
* **ステータスの進行**:
  * ボタン一つで次のステータス（例：未対応 → 面談済み）へ進める。
* **ワンクリック更新機能**:
  * ステータスが「完了（最終ステップ）」に到達した際、行の横に「次回更新」ボタンが出現。
  * クリックすると、`TaskTemplates`の計算ルールに従って「次の開始日・終了日」が自動計算され、現在のタスクは完了ログへ移動、新たなタスクが生成される。

## 4. 退所者管理
* 利用者データに `isActive` (boolean) を持たせる。
* 退所処理を行うと `isActive = false` となる。
* アラート判定やダッシュボードのクエリでは常に `WHERE is_active = true` を適用し、退所者のタスクはシステム上から隠蔽する。
* 別途「退所者アーカイブ」ページを用意し、そこではすべての過去履歴を閲覧可能とする。
