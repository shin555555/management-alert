# コマンド概要
`/sync` コマンドは、セッション終了時やキリの良いタイミングで、ドキュメント類の完了・未完了状態を「現在のソースコードの実際の状態」に一致させる保守用ワークフローです。
このコマンドにより、「コードは書いたがロードマップにチェックを入れ忘れた」といった事態を防ぎ、次回の `/onboard` を正確なものにします。

# 実行手順

1. `c:\Users\user\Desktop\management-alert\docs\04_development_roadmap.md` および `c:\Users\user\Desktop\management-alert\docs\開発進捗.md` を読み込み、現在の進捗状況、未チェック（`[ ]`）の項目、これまでの経緯を把握する。
2. これまでに作成・変更した実装ファイル（例：`src/`内のコンポーネント、Prismaのスキーマファイルなど）を必要に応じて参照し、本当にその機能が存在するか・実装が完了しているか確認する。
3. 機能が完成しているにも関わらず未チェックになっている場合、`replace_file_content` ツールを使用して `04_development_roadmap.md` のチェックボックスを `[x]` に更新する。
4. 今回の作業で決定した仕様や、次に持ち越す課題などがあれば、`replace_file_content` ツールを使用して `開発進捗.md` の履歴セクションに日付と内容を追記する。
5. ドキュメント（要件定義書や各種設計書）に追記漏れの決定事項があれば、各ファイルにも関連する追記を行う。
6. ユーザーに対し、「ドキュメントとコードの同期が完了し、ロードマップと開発進捗が最新の状態にアップデートされました」と報告すること。
