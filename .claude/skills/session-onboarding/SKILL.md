# Session Onboarding Skill
 
## 概要
 
新しいセッション開始時に、プロジェクトドキュメントを自動検出・読み込みし、AIがプロジェクトの目的・制約・現在の進捗を完全に把握するためのスキル。ユーザーが前提条件を再説明する手間を省き、すぐに作業を再開できる状態を作る。
 
---
 
## 実行ワークフロー
 
### Step 1: ドキュメント設定ファイルの確認（最優先）
 
まず、プロジェクトルートに `.onboard.md`（または `.onboard-config.md`）が存在するか確認する。
 
```
view .onboard.md
```
 
**`.onboard.md` が存在する場合**: そこに記載されたファイルリストのみを読み込む（Step 2をスキップ）。
 
**`.onboard.md` が存在しない場合**: Step 2の自動検出に進む。
 
---
 
### Step 2: ドキュメントの自動検出と読み込み
 
以下の優先順位でファイルを探し、存在するものをすべて読み込む。
 
#### 優先度 A: 要件・仕様（プロジェクトの「何を作るか」）
 
以下のパターンに合致するファイルを検索する:
 
```
docs/implementation_plan.md
docs/spec.md
docs/requirements.md
docs/README.md
SPEC.md
REQUIREMENTS.md
README.md
仕様書.md
要件定義.md
```
 
#### 優先度 B: タスク・進捗（プロジェクトの「今どこにいるか」）
 
```
docs/task.md
docs/tasks.md
docs/progress.md
docs/開発進捗.md
TASK.md
TASKS.md
TODO.md
進捗.md
タスク.md
```
 
#### 優先度 C: ルール・スタイルガイド（AIの「どう振る舞うか」）
 
```
docs/style.md
docs/rules.md
.claude/rules/
共通文体/
style-guide.md
文体分析*.txt
*.style.md
```
 
#### 優先度 D: プロジェクトメタ情報
 
```
.claude/
package.json（name/descriptionのみ）
pyproject.toml（name/descriptionのみ）
```
 
**読み込み時の注意**: ファイルが見つからない場合はスキップし、次のファイルへ進む。エラーで止まらないこと。
 
---
 
### Step 3: コンテキストの構築
 
読み込んだすべてのドキュメントから、以下を把握する:
 
1. **プロジェクトの目的**: 何を作っているか / 何のためのプロジェクトか
2. **現在の状態**: どこまで完了しているか
3. **次のタスク**: 未完了タスクのうち、最も優先度の高いもの（チェックリストなら最初の未チェック項目）
4. **厳守すべき制約**: コーディングルール、文体ルール、禁止事項など
 
---
 
### Step 4: ユーザーへの報告
 
以下のフォーマットで報告する。**簡潔に、かつ必要な情報を漏らさず**まとめること。
 
```
【プロジェクト把握完了】
 
📋 プロジェクト: [プロジェクト名・概要を1行で]
 
✅ 読み込んだドキュメント:
- [ファイル名]: [内容の要約を1行で]
- ...
 
📍 現在の状態:
[進捗の要約 / どのフェーズにいるか]
 
🎯 次に着手すべきタスク:
「[タスクの内容]」
 
ここから再開してよろしいでしょうか？
```
 
ファイルが見つからなかった場合は正直に報告し、`.onboard.md` の作成を提案する（後述）。
 
---
 
## `.onboard.md` の作成支援
 
ユーザーが「このプロジェクト用にオンボード設定を作りたい」と言った場合、または初回実行でドキュメントが見つからなかった場合は、`.onboard.md` の作成を提案する。
 
### `.onboard.md` のテンプレート
 
```markdown
# プロジェクトオンボード設定
 
## 読み込みファイルリスト（優先度順に記載）
 
### 仕様・要件
- docs/implementation_plan.md
- docs/spec.md
 
### タスク・進捗
- docs/task.md
- docs/progress.md
 
### ルール・スタイルガイド
- docs/style.md
- .claude/rules/
 
## プロジェクト概要（AIへの補足情報）
 
[このプロジェクトについてAIに知っておいてほしいこと、ファイルに書かれていない暗黙のルールなど]
```
 
---
 
## 応用パターン
 
### パターン1: 毎回のセッション開始に自動実行
 
ユーザーが新しい会話を始めて最初のメッセージが曖昧な場合（「続けて」「どこまでだっけ」など）、このスキルを自動的に実行する。
 
### パターン2: チームでの引き継ぎ
 
別の担当者や別のAIセッションへの引き継ぎ時に、`.onboard.md` をガイドとして使う。
 
### パターン3: 長期プロジェクトの定期チェックイン
 
週次・月次で「今のプロジェクト状態を把握して」と言われた場合にも同様に実行する。
 
---
 
## 注意事項
 
- **ファイルが見つからなくても止まらない**: できる範囲で読み込んで報告する
- **タスクリストの解釈**: `- [ ]` は未完了、`- [x]` は完了として扱う。番号付きリストの場合は最初の未着手項目を次タスクとする
- **複数言語対応**: 日本語・英語どちらのプロジェクトにも対応する
- **報告は簡潔に**: ドキュメントの内容を全部繰り返さない。要点と次のアクションだけ伝える
