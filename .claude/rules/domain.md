# ドメイン知識：就労継続支援A型事業所

## システムの目的

就労継続支援A型事業所が管理すべき行政書類・内部書類の**更新期限を一元管理**し、
更新漏れを防ぐためのシステム。現場スタッフが毎日確認するダッシュボードが中心。

## 主要エンティティ

| エンティティ | 日本語 | 説明 |
|------------|------|------|
| `Facility` | 事業所 | A型事業所（マルチテナント対応を想定） |
| `User` | スタッフ | 事業所のスタッフ。ADMIN / STAFF の2ロール |
| `Client` | 利用者 | 事業所を利用する障害者 |
| `TaskTemplate` | タスクテンプレート | 「支給決定期間」など管理すべき書類のマスタ |
| `ClientTask` | クライアントタスク | 利用者ごとの実際の期限・ステータス |
| `TaskHistory` | タスク履歴 | ステータス変更の監査証跡 |

## デフォルト管理項目（A型事業所基本セット）

| No. | 項目名 | カテゴリ | 計算パターン | 周期 |
|----|--------|--------|------------|-----|
| 1 | 支給決定期間 | 行政手続き | MONTH_END | 1年 |
| 2 | 利用者負担の適用期間 | 行政手続き | MONTH_END | 1年 |
| 3 | 在宅利用期間 | 行政手続き | ADD | 6ヶ月 |
| 4 | 障害者手帳の有効期限 | 行政手続き | MANUAL | 手動 |
| 5 | 個別支援計画 | 内部書類 | REPEAT | 6ヶ月ごと |
| 6 | モニタリング | 内部書類 | REPEAT | 6ヶ月ごと |

## 計算パターン（CalculationPattern）

| パターン | 説明 | 例 |
|--------|------|-----|
| `ADD` | 開始日から N ヶ月/年後 | 6ヶ月後 |
| `FIXED` | 毎年特定の日付 | 毎年4月1日 |
| `REPEAT` | 開始日から N ヶ月ごとに繰り返し | 6ヶ月ごと |
| `MONTH_END` | N 年(ヶ月)後の月末 | 1年後の月末 |
| `MANUAL` | ルールなし・手動入力 | 障害者手帳など |

## アラートレベル

期限までの残り日数に応じて色分け表示する：

| レベル | 色 | デフォルト閾値 | 意味 |
|-------|---|------------|-----|
| `yellow` | 黄 | 6週間前 | 要注意 |
| `orange` | 橙 | 4週間前 | 対応開始 |
| `red` | 赤 | 2週間前 | 緊急対応 |
| `overdue` | 濃赤 | 期限超過 | 超過 |
| `none` | — | 閾値外 | 問題なし |

閾値はテンプレートごとに `alertSteps` JSON で自由に設定可能。

## ステータスフロー

各タスクは `statusFlow` に定義されたステータスを順番に進む。

例（個別支援計画）:
```
未対応 → 面談済み → 書類作成済み → 署名・押印済み（完了・更新）
```

- 最後のステータス（末尾）が「完了」扱い
- 完了後に「ワンクリック更新」ボタンが現れ、次回期限タスクを自動生成する

## 重要な業務用語

| 用語 | 意味 |
|-----|-----|
| 受給者証 | 障害福祉サービスを受けるための証書。有効期限がある |
| 支給決定期間 | 受給者証に記載された有効期間（通常1年） |
| 個別支援計画 | 利用者ごとの支援目標・内容を定めた書類（6ヶ月ごとに見直し） |
| モニタリング | 個別支援計画の進捗確認（6ヶ月ごと） |
| 退所 | 利用者が事業所を辞めること。`isActive = false` になる |
| アーカイブ | 退所者データ。削除はせず全履歴を保持（監査対策） |
| 爆速入力UI | 和暦直接入力・ワンクリック全選択・↑↓キー操作を備えた日付入力 |
| スマート予測入力 | 新規登録時に利用開始月を選ぶだけで全タスクの期限を自動計算する機能 |
| ワンクリック更新 | タスク完了時に「次回」タスクを1クリックで自動生成する機能 |
